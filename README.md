This repository contains scripts and data for collecting and processing health metrics from Transcend TS128GSSD450K solid-state drives (SSDs) used in Locus chassis. The primary purpose is to parse raw data from various sources and consolidate key performance and health indicators into a single, analyzable CSV file.

How It Works
The data processing pipeline is orchestrated by main.py and follows these steps:

Chassis and SSD Mapping: The script first reads Data/Raw/Locus-Chassis-SSD.xlsx, which contains the mapping between a chassis serial number and the serial number of the SSD installed within it.
Initial CSV Creation: Using this mapping, it creates a new file at Data/Processed/collected_data.csv, populating it with the Chassis Serial No. and SSD Serial No. columns. This file is overwritten on each run to ensure fresh data.
S.M.A.R.T. Data Extraction: The script then iterates through the CSV files located in Data/Raw/Transcend-data/. Each file in this directory is an export from Transcend's SSD Scope software and is named after the SSD's serial number.
Data Consolidation: For each SSD serial number found in collected_data.csv, the script parses the corresponding file from the Transcend-data directory to extract the following S.M.A.R.T. attributes:
Power-On Hours
Number of Valid Spare Blocks
Remain Life (percentage)
Ultra DMA CRC Error Count
Total LBA Written
Final Output: The extracted health metrics are written into the appropriate rows and columns of Data/Processed/collected_data.csv, resulting in a complete, aggregated dataset.
File Structure
.
├── main.py
├── Data/
│   ├── Processed/
│   │   └── collected_data.csv  # Final aggregated output
│   └── Raw/
│       ├── Locus-Chassis-SSD.xlsx # Maps chassis to SSD serial numbers
│       ├── Smartctl/              # Raw smartctl command outputs
│       └── Transcend-data/        # Raw S.M.A.R.T. data exports from SSD Scope
└── src/
    ├── create_file_names.py
    ├── get_serial_numbers.py
    └── get_transcend_data.py
main.py: The entry point for executing the entire data collection and processing workflow.
Data/Raw/: Contains all original, unprocessed source data.
Data/Processed/: Home for the final, clean CSV file generated by the script.
src/: Contains the Python modules responsible for the processing logic.
Getting Started
Prerequisites
Python 3.x
Pandas
Openpyxl
Installation
Clone the repository:
git clone https://github.com/fortunatah/Locus-SSD-Data.git
cd Locus-SSD-Data
Install the required Python packages:
pip install pandas openpyxl
Execution
To run the script and generate the final dataset, execute main.py from the root directory of the repository:

python main.py
The script will print "Collecting data...." and, upon completion, the updated collected_data.csv file will be available in the Data/Processed/ directory.

Output Data Schema
The final output file Data/Processed/collected_data.csv contains the following columns:

| Column Name | Description | | -------------------- | --------------------------------------------------------------------------- | | Chassis Serial No. | The serial number of the Locus chassis. | | SSD Serial No. | The serial number of the Transcend SSD. | | Power-On Hours | The total number of hours the drive has been powered on. | | Valid Blocks | The number of valid spare blocks remaining on the drive. | | Life Percentage | An estimate of the SSD's remaining lifespan, as a percentage. | | CRC | The count of Ultra DMA CRC errors, indicating data transfer issues. | | Total LBA Written | The total amount of data written to the drive (unit from raw data is 32MB). |
